# 养生知识库系统 v2.0

## 简介

这是一个可扩展的养生知识库系统，支持多种类型的内容：
- 🌿 食疗方子
- 👆 穴位按摩
- 🧘 养生功法
- 📖 健康知识

## 架构特点

### 1. 可扩展
- 支持不断增加的养生内容
- 支持多位作者的书籍
- 支持多种内容类型

### 2. 自适应
- 数据自动索引
- 分类自动生成
- 筛选条件动态调整

### 3. 易维护
- 数据与代码分离
- 统一的数据格式
- 清晰的文件结构

## 使用方法

### 1. 启动网站
```bash
# 方式一：使用v1版本（当前稳定版）
# 直接打开 index.html

# 方式二：使用v2版本（新架构）
# 将 index_v2.html 重命名为 index.html
# 将 app_v2.js 重命名为 app.js
```

### 2. 添加新数据

#### 添加食疗方子
```bash
# 将新数据放入 data/ 目录
# 运行合并脚本
python3 scripts/merge_data.py
```

#### 添加穴位数据
```json
// data/acupoints.json
{
  "version": "1.0",
  "items": [
    {
      "id": "acupoint-001",
      "name": "内关穴",
      "location": "腕横纹上2寸，掌长肌腱与桡侧腕屈肌腱之间",
      "efficacy": "宁心安神、理气止痛",
      "method": "用拇指按压，每次3-5分钟",
      "symptoms": ["心悸", "失眠", "胸闷"]
    }
  ]
}
```

#### 添加功法数据
```json
// data/exercises.json
{
  "version": "1.0",
  "items": [
    {
      "id": "exercise-001",
      "name": "八段锦",
      "description": "传统养生功法",
      "steps": ["两手托天理三焦", "左右开弓似射雕", ...],
      "duration": "15-20分钟",
      "frequency": "每天1-2次"
    }
  ]
}
```

### 3. 更新索引
添加新数据后，需要更新 `data/index.json`：
```json
{
  "stats": {
    "total": 800,  // 更新总数
    "byType": {
      "recipe": 795,
      "acupoint": 5,  // 新增
      "exercise": 0
    }
  }
}
```

## 数据格式规范

### 通用字段
```json
{
  "id": "唯一标识",
  "name": "名称",
  "type": "内容类型",
  "source": {
    "book": "来源书籍",
    "author": "作者"
  },
  "tags": ["标签"],
  "symptoms": ["相关症状"],
  "season": ["适用季节"]
}
```

### 类型特定字段

#### 食疗方子 (recipe)
```json
{
  "ingredients": [{"name": "", "amount": ""}],
  "method": "制作方法",
  "precautions": "注意事项",
  "suitableFor": ["适用人群"],
  "contraindicatedFor": ["禁忌人群"]
}
```

#### 穴位 (acupoint)
```json
{
  "location": "位置描述",
  "method": "按摩方法",
  "duration": "按摩时长",
  "image": "穴位图（可选）"
}
```

#### 功法 (exercise)
```json
{
  "steps": ["步骤1", "步骤2"],
  "duration": "练习时长",
  "frequency": "练习频率",
  "benefits": "益处"
}
```

## 扩展指南

### 添加新作者
1. 在 `data/index.json` 的 `sources` 数组中添加新作者信息
2. 创建对应的数据文件
3. 更新统计数据

### 添加新内容类型
1. 在 `data/index.json` 的 `categories.types` 中添加新类型
2. 创建对应的数据文件
3. 在 `app_v2.js` 中添加对应的渲染函数
4. 更新 `index_v2.html` 中的类型选择器
